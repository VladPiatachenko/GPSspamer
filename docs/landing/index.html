Симулятор GPS spoofing атак

Бакалаврська робота, присвячена розробці сервісу для отримання реальної телеметрії польоту та моделювання атак підміни координат з метою дослідження алгоритмів виявлення GPS-spoofing.

Назва (UA): Симулятор GPS spoofing атак для дослідження виявлення підміни координат
Title (EN): GPS Spoofing Simulation Service for Detection Research

Короткий опис: Сервіс GPSspamer отримує реальні координати за flightId, застосовує атаки BIAS/DRIFT/NOISE та формує логи truth vs attacked для дослідження методів виявлення GPS-spoofing.

Ключові слова: GPS spoofing, UAV, telemetry, Spring Boot, anomaly detection

Актуальність теми

GPS spoofing є однією з ключових загроз для навігаційних систем БПЛА та літальних апаратів. Для створення ефективних алгоритмів захисту необхідні контрольовані дані з відомими параметрами атак.

Мета дослідження

Розробити програмний сервіс для симуляції GPS spoofing атак та формування розмічених наборів даних truth vs attacked для аналізу та тестування алгоритмів виявлення.

Основні завдання

  1. Отримання телеметрії за ідентифікатором польоту (flightId).
  2. Реалізація моделей атак: BIAS, DRIFT, NOISE.
  3. Паралельна обробка істинних та атакованих координат.
  4. Потокова передача даних (SSE) для спостереження у реальному часі.
  5. Журналювання даних у CSV для офлайн-аналізу.

Типи атак

  • BIAS — сталий зсув у метрах.
  • DRIFT — дрейф з часом (м/с).
  • NOISE — випадковий шум із sigma (метри).

Джерело даних телеметрії

Отримання реальних координат здійснюється за flightId через інтеграцію з зовнішнім сервісом моніторингу авіаруху (наприклад, FlightRadar24). Це дозволяє відстежувати реальний рух об’єкта та накладати контрольовані атаки.

Приклади атак і траєкторій

Нижче наведено приклади експериментальних траєкторій (істинні дані та підміна), використані в публікації: Visnyk KrNU, 2025 (PDF) . Зеленим позначено нормальну траєкторію, червоним — точки після GPS-spoofing.

Дані та датасети зі статті

  • Реальні траєкторії: телеметрія за flightId (інтеграція з FR24/аналогом).
  • Синтетичні атаки: BIAS / DRIFT / NOISE з параметрами (метри, м/с, sigma).
  • Датасети (з публікації): додай тут точні назви наборів даних з PDF (2–4 пункти).

Реалізований програмний застосунок

Веб-інтерфейс дозволяє обрати flightId, налаштувати тип атаки (BIAS / DRIFT / NOISE), спостерігати різницю між істинними та атакованими координатами, а також формувати CSV-лог для подальшого аналізу.

Очікувані результати

  • Функціональний backend-сервіс для симуляції GPS spoofing атак у реальному часі.
  • Генерація розмічених даних (truth vs attacked) для офлайн-досліджень.
  • Інтерфейс інтеграції з системами виявлення атак (REST/SSE).
